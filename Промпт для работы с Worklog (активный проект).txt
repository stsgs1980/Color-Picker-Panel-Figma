# Промпт для работы с Worklog (активный проект)

> **Использование:** Вставьте этот промпт при вызове любого подагента для обеспечения корректной работы с ворклогом.

---

## 🔧 Полный промпт для подагентов

```markdown
## ⚠️ ОБЯЗАТЕЛЬНЫЕ ТРЕБОВАНИЯ К РАБОТЕ С WORKLOG

### 1. Перед началом работы

Ты ОБЯЗАН прочитать файл worklog перед выполнением любой задачи:

```
/home/z/my-project/worklog.md
```

Это необходимо для:
- Понимания контекста предыдущих задач
- Избежания дублирования работы
- Преемственности в принятии решений

### 2. Твой Task ID

Твой Task ID: **{TASK_ID}**

Используй этот идентификатор во всех записях.

### 3. После завершения работы

Ты ОБЯЗАН добавить запись в worklog.md в следующем формате:

```markdown
---
Task ID: {TASK_ID}
Agent: {AGENT_NAME}
Task: {Краткое описание задачи}

Work Log:
- {Конкретное действие 1}
- {Конкретное действие 2}
- {Конкретное действие 3}

Stage Summary:
- Files created: {список файлов}
- Files modified: {список файлов}
- Key decisions: {важные решения}
- Status: completed
```

### 4. Критически важные правила

| Правило | Описание |
|---------|----------|
| ❌ НЕ ПЕРЕЗАПИСЫВАТЬ | Только добавлять в конец файла |
| ✅ ИСПОЛЬЗОВАТЬ РАЗДЕЛИТЕЛЬ | Каждая запись начинается с `---` |
| ✅ УКАЗЫВАТЬ ФАЙЛЫ | Перечислять все созданные/изменённые файлы |
| ✅ ФИКСИРОВАТЬ РЕШЕНИЯ | Документировать важные технические решения |

### 5. Пример корректной записи

```markdown
---
Task ID: 2-a
Agent: full-stack-developer
Task: Создание API endpoints для аутентификации

Work Log:
- Создан /app/api/auth/login/route.ts (POST endpoint)
- Создан /app/api/auth/register/route.ts (POST endpoint)
- Добавлена валидация через Zod схемы
- Интегрирован Prisma для работы с БД
- Написаны unit-тесты (покрытие 89%)

Stage Summary:
- Files created: src/app/api/auth/login/route.ts, src/app/api/auth/register/route.ts, src/__tests__/auth.test.ts
- Files modified: prisma/schema.prisma, src/lib/validators/auth.ts
- Key decisions: Использован JWT для токенов, bcrypt для хеширования паролей
- Status: completed
```

### 6. Алгоритм работы

```
1. Прочитать /home/z/my-project/worklog.md
      ↓
2. Изучить контекст предыдущих задач
      ↓
3. Выполнить поставленную задачу
      ↓
4. Зафиксировать действия (Work Log)
      ↓
5. Добавить запись в конец worklog.md
      ↓
6. Подтвердить завершение
```

---

## Твоя задача

{ОПИСАНИЕ_ЗАДАЧИ}

---
```

```javascript
Task({
  description: "Создание API пользователей",
  prompt: `

Before starting: READ \`/home/z/my-project/worklog.md\`

Your Task ID: **2-a**

After completion: APPEND to worklog.md (do NOT overwrite!)

Format:
---
Task ID: 2-a
Agent: full-stack-developer
Task: {description}

Work Log:
- {actions}

Stage Summary:
- Files: {list}
- Decisions: {list}
- Status: completed

---

## Твоя задача

Создай REST API для управления пользователями:
- GET /api/users — список пользователей
- POST /api/users — создание пользователя
- GET /api/users/[id] — получение по ID
- PUT /api/users/[id] — обновление
- DELETE /api/users/[id] — удаление

Используй Prisma ORM и валидацию Zod.
`,
  subagent_type: "full-stack-developer"
});
```

## 🔄 Промпт для продолжения работы (после паузы)

```markdown
## ПРОДОЛЖЕНИЕ РАБОТЫ НАД ПРОЕКТОМ

### Шаг 1: Анализ текущего состояния

1. Прочитай `/home/z/my-project/worklog.md`
2. Определи последний завершённый Task ID
3. Изучи контекст и принятые решения

### Шаг 2: Определение следующего шага

На основе прочитанного:
- Какие задачи уже выполнены?
- Какая задача следующая в очереди?
- Есть ли незавершённые подзадачи?

### Шаг 3: Продолжение работы

Твой Task ID: **{TASK_ID}** (определяется на основе предыдущих)

После завершения — добавить запись в worklog.md.

### Текущая задача:

{ОПИСАНИЕ_ЗАДАЧИ}
```

## 🛠 Промпт для Super Z (главный агент)

```markdown
## ПРАВИЛА РАБОТЫ С WORKLOG ДЛЯ ГЛАВНОГО АГЕНТА

### При вызове подагентов

ОБЯЗАТЕЛЬНО включай в каждый промпт:

1. **Task ID** — уникальный идентификатор задачи
2. **Инструкцию по чтению** worklog.md перед работой
3. **Инструкцию по записи** после работы

### Шаблон промпта для подагента

\`\`\`
Task ID: {ID}

ВАЖНО:
1. Перед работой прочитай /home/z/my-project/worklog.md
2. После работы ДОБАВИТЬ (не перезаписать!) запись в worklog.md

Формат записи:
---
Task ID: {ID}
Agent: {AGENT_TYPE}
Task: {описание}

Work Log:
- {действия}

Stage Summary:
- {результаты}
---

Задача: {DESCRIPTION}
\`\`\`

### После завершения работы подагента

1. Проверить, что запись в worklog.md добавлена
2. Обновить TODO-лист (статус: completed)
3. При необходимости — запустить следующий агент

### Контроль качества записи

Проверять наличие:
- [ ] Task ID в правильном формате
- [ ] Имя агента
- [ ] Описание задачи
- [ ] Work Log с действиями
- [ ] Stage Summary с файлами и решениями
```
